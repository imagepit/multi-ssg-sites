## 🚫 はじめに

プロンプト設計の基礎技術を学んだあなたは、次に「出力品質をさらに精密にコントロールする」技術が必要です。このページでは、**否定プロンプト（NG指定）**という強力な制御技術を学び、v0の生成精度を劇的に向上させる方法を習得します。

否定プロンプトは、「何を作るか」と同じくらい「**何を作らないか**」を明確に指示する技術です。適切な否定プロンプト設計により、意図しないデザインパターンや機能を排除し、より質の高いUIを生成できるようになります。

### このページで学べる事

否定プロンプトの理論と実践的な活用方法を体系的に学び、v0の生成品質を次のレベルへ引き上げます。

:::note

- 否定プロンプトの基本概念と有効性の理解
- 避けたいデザインパターンの具体的な指定方法
- 正の指示と負の指示の最適なバランス技術
- 実務で役立つ否定プロンプトのパターンと実践演習
- 品質向上のためのベストプラクティス

:::

## 🔍 否定プロンプトの基本を理解する

### 否定プロンプトとは何か？

否定プロンプトとは、AIに**生成してほしくない要素やパターンを明示的に指定する**指示のことです。従来の「作ってほしいもの」を指示する正のプロンプトに対し、「作ってほしくないもの」を指定する負のプロンプトです。

:::note 否定プロンプトの重要性

否定プロンプトが重要な理由は、予防的な品質制御が可能になるためです。問題が発生する前に予防策を講じることで、手戻りを削減し、開発効率を大幅に向上させることができます。

:::

**否定プロンプトが効果的な3つのシーン**:

1. **デザインの一貫性維持**: ブランドガイドラインに反する要素を排除
2. **技術的制約の遵守**: 使用できない技術やアプローチを事前に指定
3. **品質基準の保証**: アクセシビリティやパフォーマンス要件を満たさない要素を防止

### 基本プロンプトとの比較

まずは、基本プロンプトだけではどのような問題が発生するかを見てみましょう。

**基本プロンプトのみのケース**:
```bash
モダンなユーザープロフィールカードを作成してください
```

このプロンプトでは、以下のような問題が発生する可能性があります：

- 望ましくない色使いやフォントが使用される
- レスポンシブ対応が不十分
- アクセシビリティを考慮していない
- ブランドガイドラインに違反する要素が含まれる

**否定プロンプトを追加したケース**:
```bash
モダンなユーザープロフィールカードを作成してください。ただし、以下の要素は使用しないでください：
- 丸みを帯びたデザイン要素
- グラデーション背景
- 中央揃えのレイアウト
- サイズが大きすぎるアバター画像
- 不要なアニメーション効果
```

このように、否定プロンプトを追加することで、生成結果の品質と一貫性を大幅に向上させることができます。

## 🎯 否定プロンプトの具体的な活用シーン

実際の開発現場で役立つ否定プロンプトの活用パターンを学びましょう。

### デザインパターンの制御

最も一般的な使用シーンです。ブランドガイドラインに沿ったデザインを確保します。

:::syntax デザイン制御の否定プロンプト

```bash
[目的] ただし、以下の要素は使用しないでください：
- [避けたい色1]、[避けたい色2]などの色彩
- [避けたい形状]のデザイン要素
- [避けたいレイアウト]パターン
```

目的を明確にした上で、避けたいデザイン要素を具体的に指定します。

:::

**実践例**:

```bash
Eコマースサイトの商品カードを作成してください。ただし、以下の要素は使用しないでください：
- ネオンカラーや蛍光色
- 丸みを帯びたボタンデザイン
- 中央揃えのレイアウト
- 装飾的なアイコンやイラスト
```

### 機能制限の指定

特定の機能を排除したい場合に使用します。

:::syntax 機能制限の否定プロンプト

```bash
[目的]を作成してください。ただし、[機能名]機能は実装しないでください。
理由：[なぜその機能を避けるのかの説明]
```

機能を制限する理由を明確にすることで、AIがより適切な代替案を提案しやすくなります。

:::

**実践例**:

```bash
ユーザー登録フォームを作成してください。ただし、ソーシャルログイン機能は実装しないでください。
理由：プライバシーポリシーにより、外部サービス連携を制限しているため
```

### 技術選択の制御

使用しない技術やアプローチを明確にします。

:::syntax 技術制御の否定プロンプト

```bash
[目的]を実装してください。ただし、以下の技術は使用しないでください：
- [技術1]
- [技術2]
- [技術3]

代わりに、[推奨技術]を使用してください。
```

使用しない技術を明確にし、代替の推奨技術を提示します。

:::

**実践例**:

```bash
データ表示テーブルを作成してください。ただし、以下の技術は使用しないでください：
- jQuery
- BootstrapのGridシステム
- テーブルレイアウト（tableタグ）

代わりに、CSS GridとFlexboxを使用してください。
```

## ⚖️ 正の指示と負の指示のバランス

効果的なプロンプト設計では、正の指示と負の指示のバランスが重要です。

### バランスの黄金比

経験則として、**「正の指示7：負の指示3」**の比率が効果的です。

:::note プロンプトバランスの原則

正の指示で「何を作るか」を明確にし、負の指示で「品質を保護する」。負の指示が多すぎると、AIの創造性が制限されすぎてしまう可能性があります。

:::

**バランスの良いプロンプト例**:

```bash
モバイルファーストのレスポンシブなタスクリストアプリを作成してください。
✔️ 正の指示：
- タスクの追加、編集、削除機能
- ドラッグ＆ドロップでの並び替え
- 進捗状況の視覚化
- 明るいクリーンなデザイン

❌ 負の指示：
- ダークモードは実装しない
- 複雑なアニメーションは避ける
- ポップアップ通知は使用しない
```

### バランスが悪いケース

負の指示が多すぎるプロンプトの問題点：

```bash
タスクリストアプリを作成してください。ただし、以下の要素はすべて避けてください：
- ダークモード、アニメーション、ポップアップ
- 丸いボタン、影、グラデーション
- 複雑な状態管理、副作用、再レンダリング
- （以下も同様に続く...）
```

このようなプロンプトでは、AIが何を作れば良いのか混乱し、かえって品質が低下します。

## 🛡️ 品質保証のための否定プロンプト

品質保証の観点から、特に重要な否定プロンプトの活用方法を学びます。

### アクセシビリティ制約の指定

アクセシビリティを確保するための重要な制約です。

:::syntax アクセシビリティ否定プロンプト

```bash
[目的]を作成してください。ただし、以下のアクセシビリティ制約を厳守してください：
- 色だけで情報を伝えるデザインは避ける
- キーボード操作が不可能な要素は作成しない
- 拡大縮小に対応しない固定サイズは使用しない
```

アクセシビリティを確保するための具体的な制約を指定します。

:::

**実践例**:

```bash
データ可視化ダッシュボードを作成してください。ただし、以下のアクセシビリティ制約を厳守してください：
- 色だけでデータの区別をしない（テクスチャやパターンも併用）
- マウスホーバーだけで表示される情報は作成しない
- 固定幅のレイアウトは避け、相対単位を使用する
```

### パフォーマンス制約の指定

パフォーマンスを考慮した制約を追加します。

:::syntax パフォーマンス否定プロンプト

```bash
[目的]を実装してください。ただし、以下のパフォーマンス制約を考慮してください：
- 重量の大きな画像は直接使用しない
- レンダリングをブロックする処理は避ける
- 不要な再レンダリングを発生させる実装は禁止
```

パフォーマンスを考慮した実装制約を指定します。

:::

**実践例**:

```bash
画像ギャラリーページを作成してください。ただし、以下のパフォーマンス制約を考慮してください：
- ページ読み込み時にすべての画像を読み込まない
- スクロールに応じた遅延読み込みを実装する
- 画像サイズの最適化を行う
```

## 🎨 否定プロンプトを使ったUI生成を動かして確認してみよう

それでは、学習した否定プロンプト技術を実際に試してみましょう。基本プロンプトとの比較を通じて、品質向上効果を実感します。

:::step

1. 作業用プロジェクトの準備

任意の場所（デスクトップなど）で`negative-prompt-test`フォルダを作成し、VSCodeでそのフォルダを開いてください。

2. 基本プロンプトでのUI生成

まずは基本プロンプトのみでUIを生成してみましょう。v0で以下のプロンプトを実行してください。

```bash
モダンなユーザープロフィールカードを作成してください
```

**生成結果の確認ポイント**:
- 予期せぬデザイン要素が含まれていないか
- レスポンシブ対応は適切か
- アクセシビリティを考慮しているか

3. 否定プロンプトを追加したUI生成

次に、否定プロンプトを追加して同じUIを生成します。

```bash
モダンなユーザープロフィールカードを作成してください。ただし、以下の要素は使用しないでください：
- 丸みを帯びたデザイン要素
- グラデーション背景
- 中央揃えのレイアウト
- サイズが大きすぎるアバター画像
- 不要なアニメーション効果
```

4. 2つの結果を比較

生成された2つのUIを比較し、以下の点を評価してください：

- 否定プロンプトにより、意図しない要素が排除されているか
- 全体的なデザインの一貫性が向上しているか
- ブランドガイドラインに沿ったデザインになっているか

5. ブランド制約付きの否定プロンプト

さらにブランド制約を追加したプロンプトで生成してみましょう。

```bash
企業向けユーザープロフィールカードを作成してください。
要件：
- 氏名、役職、部署、連絡先を表示
- プロフェッショナルな印象を与えるデザイン
- モバイル対応

ただし、以下の要素は使用しないでください：
- カジュアルなカラーパレット（明るい色やパステルカラー）
- 装飾的なアイコンやイラスト
- 丸みを帯びたボタンやカード
- 影の強いデザイン
- 中央揃えのレイアウト（左揃えを基本とする）
```

6. 生成結果の検証

最終的な生成結果が、ビジネス向けプロフェッショナルなデザインになっているかを確認してください。

:::

このように否定プロンプトを使用することで、意図した品質のUIをより確実に生成できるようになります。

## 📊 否定プロンプトの効果的なパターン集

実務で役立つ否定プロンプトのパターンを体系的にまとめました。

### デザイン系パターン

**カラーパレット制御**:
```bash
[目的]を作成してください。ただし、以下の色は使用しないでください：
- #FF0000系の赤色
- 蛍光色やネオンカラー
- コントラストが低い色の組み合わせ
```

**レイアウト制御**:
```bash
[目的]を実装してください。ただし、以下のレイアウトパターンは避けてください：
- 中央揃えのコンテンツ配置
- 不均等なスペーシング
- 混沌とした要素配置
```

**タイポグラフィ制御**:
```bash
[目的]を作成してください。ただし、以下のタイポグラフィ制約を守ってください：
- 装飾的なフォントは使用しない
- フォントサイズの不統一を避ける
- 行間が詰まりすぎたテキストは作成しない
```

### 機能系パターン

**インタラクション制御**:
```bash
[目的]を実装してください。ただし、以下のインタラクションは実装しないでください：
- ホバー時の予期せぬ状態変化
- 複雑なアニメーション
- ユーザーを混乱させるようなフィードバック
```

**データ表示制御**:
```bash
[目的]を作成してください。ただし、以下のデータ表示パターンは避けてください：
- 情報過多な表示
- 重要な情報を隠すデザイン
- 理解しにくいデータ形式
```

### 技術系パターン

**パフォーマンス制御**:
```bash
[目的]を実装してください。ただし、以下のパフォーマンス問題を引き起こす実装は避けてください：
- 不必要な再レンダリング
- 大量のDOM操作
- 最適化されていない画像読み込み
```

**保守性制御**:
```bash
[目的]を作成してください。ただし、以下の保守性問題を引き起こすコードは書かないでください：
- コメントのない複雑なロジック
- マジックナンバーの使用
- 責務が不明確なコンポーネント設計
```

## ⚠️ 否定プロンプトの注意点とベストプラクティス

否定プロンプトを効果的に使用するための重要な注意点を学びます。

### 過剰な否定制御のリスク

:::warning 過剰制御の危険性

否定プロンプトが多すぎると、以下のような問題が発生します：
- AIの創造性が過度に制限される
- 逆説的に予期せぬ出力が増える
- プロンプトが複雑になりすぎてメンテナンスが困難になる

:::

**適切な否定制御の目安**:
- 1プロンプトあたり3-5個の否定項目が適切
- 重要な制約から優先的に指定
- 正の指示とのバランスを常に意識

### 曖昧な否定表現の問題

具体的でない否定表現は、期待した効果を得られません。

**悪い例**:
```bash
綺麗なデザインにしてください（ただし、ダメな要素は避けて）
```

**良い例**:
```bash
モダンなデザインにしてください（ただし、以下の要素は使用しないでください：
- グラデーション背景
- 丸みを帯びたボタン
- 不必要な影効果）
```

### 矛盾した指示の問題

正の指示と負の指示が矛盾しないように注意が必要です。

**矛盾例**:
```bash
モダンでクリエイティブなデザインを作成してください。
ただし、すべての要素は伝統的で保守的なスタイルにしてください。
```

このような矛盾した指示は、AIを混乱させ品質を低下させます。

## 📈 否定プロンプトの効果測定と改善

否定プロンプトの効果を客観的に評価し、継続的に改善する方法を学びます。

### 効果測定の指標

:::note 効果測定のためのKPI

**生成成功率**: 意図した通りのUIが生成される割合
**修正時間**: 生成後の手動修正にかかる時間
**一貫性スコア**: 複数回の生成で一貫性が保たれる度合い
**満足度**: チームメンバーの主観的評価

:::

### A/Bテストの実施

否定プロンプトの有無で生成結果を比較検証します。

```markdown
## A/Bテスト結果の記録例

### テストケース：ユーザープロフィールカード

**バージョンA（基本プロンプトのみ）**:
- 生成時間：30秒
- 修正時間：45分
- 満足度：3/5

**バージョンB（否定プロンプト追加）**:
- 生成時間：35秒
- 修正時間：15分
- 満足度：4.5/5

**結論**：
否定プロンプトにより修正時間が67%削減され、品質も向上
```

### 継続的な改善サイクル

1. **現状のプロンプトで生成**
2. **結果を評価し問題点を特定**
3. **適切な否定プロンプトを追加**
4. **再生成して効果を確認**
5. **成功したパターンをチームで共有**

## まとめ

否定プロンプトは、v0でのUI生成品質を劇的に向上させる強力な技術です。「何を作るか」だけでなく「**何を作らないか**」を明確にすることで、より精密な制御が可能になります。

:::note 要点のまとめ

- 否定プロンプトは予防的な品質制御技術として非常に効果的
- 正の指示7：負の指示3のバランスが理想的
- デザイン、機能、技術、品質の各側面で活用可能
- 具体的で明確な表現を使用することが重要
- 過剰な制御は逆効果になるため適度な使用を心がける
- 効果測定と継続的な改善が重要

:::

否定プロンプト技術を習得することで、あなたはv0を単なるUI生成ツールから、チームの開発プロセスに統合できる信頼性の高いパートナーへと進化させることができます。次のステップでは、これまで学んだプロンプト設計技術を実践的なプロジェクトで活用する方法を学びましょう。

**[目的→制約→受入基準の書き方](./objective-constraints-acceptance.md)**に戻って基礎を復習するか、**[差分指示（部分更新/セレクタ指定）](./diff-prompts.md)**でより高度な制御技術を学びましょう。
