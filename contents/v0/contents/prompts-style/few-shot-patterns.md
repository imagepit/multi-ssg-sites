## はじめに

v0でUIを生成する際、同じプロンプトでも毎回異なる結果が生成されてしまい、品質の一貫性を保つのが難しいと感じたことはありませんか？この問題を解決する強力な手法が「few-shot learning（例示学習）」です。

### このページで学べる事

具体的な例を示すことでv0の生成品質をコントロールするfew-shotテクニックを体系的に習得します。

:::note

- Few-shot learningの基本概念とv0での有効性
- ポジティブ例、ネガティブ例、前後比較などの例示パターン
- UIコンポーネント別の効果的な例示方法
- コードレベルでのスタイル指定技術
- 例示ライブラリの構築と管理方法
- 実際のプロジェクトでの適用ノウハウ

:::

## 🎯 Few-shot Learningとは

Few-shot learningは、AIモデルに少数の具体例を示すことで、望ましい出力スタイルや品質を学習させる手法です。v0においても、この技術は非常に効果的に機能します。

### Few-shotがv0で効果的な理由

v0は大規模言語モデルをベースとしているため、**文脈理解能力**が非常に高いです。具体的な例を示すことで、抽象的な指示よりも遥かに正確に意図を伝えることができます。

:::note Few-shot Learningとは

Few-shot Learning（少ショット学習）とは、機械学習における学習手法の一つで、少数の訓練例（ショット）から新しいタスクを学習するアプローチです。従来の機械学習が大量のデータを必要とするのに対し、Few-shotでは数個〜数十個の例で効果を発揮します。

:::

### 基本プロンプトとの比較

まず、基本プロンプトとfew-shotプロンプトの違いを見てみましょう。

**基本プロンプト：**
```
ユーザープロフィールカードを作成してください
```

**Few-shotプロンプト：**
```
ユーザープロフィールカードを作成してください。

例：
<良い例>
画像：左寄せの円形アバター（64x64px）
名前：太字、18px、グレー900
役職：通常、14px、グレー600
フォロワー数：バッジ形式、ブルー背景
アクションボタン：プライマリカラー、角丸8px
カード全体：白背景、シャドウ、パディング16px
</良い例>
```

このように具体的な例を示すことで、v0はより正確に意図を理解し、一貫性のあるUIを生成します。

## 📋 例示の種類と効果的な使い方

Few-shotプロンプトでは、目的に応じて異なる種類の例を効果的に組み合わせることが重要です。

### 1️⃣ ポジティブ例（望ましい出力の具体化）

ポジティブ例は「このように出力してほしい」という望ましい結果を具体的に示します。

:::syntax ポジティブ例の基本構造

```markdown
[コンポーネント名]を作成してください。

例：
<良い例>
[具体的なUI要素の詳細な説明]
- 見た目：色、サイズ、配置、間隔
- 機能：クリック動作、状態変化
- レスポンシブ：ブレークポイントごとの挙動
</良い例>
```

:::

**実践例：**
```
ブログ記事カードを作成してください。

例：
<良い例>
アイキャッチ画像：16:9のアスペクト比、角丸8px
タイトル：太字、20px、行高さ1.4、最大2行まで
抜粋文：通常、14px、グレー600、最大3行まで
メタ情報：日付とカテゴリー、12px、グレー500
ホバーエフェクト：画像が拡大、シャドウが濃くなる
カード全体：白背景、シャドウsm、パディング20px、マージン16px
</良い例>
```

### 2️⃣ ネガティブ例（避けるべきパターンの明示）

ネガティブ例は「このような出力は避けてほしい」という避けるべきパターンを明示します。

:::syntax ネガティブ例の基本構造

```markdown
[コンポーネント名]を作成してください。

避けてほしい例：
<悪い例>
[避けるべきUI要素の説明]
- 見た目：不適切な色、サイズ、配置
- 機能：使いにくいインタラクション
- パフォーマンス：重い処理、不要なアニメーション
</悪い例>
```

:::

**実践例：**
```
モーダルダイアログを作成してください。

避けてほしい例：
<悪い例>
画面全体を覆う半透明の背景がない
ESCキーや背景クリックで閉じられない
コンテンツが中央に配置されていない
スクロールできない状態でコンテンツが溢れる
フォーカストラップが機能していない
</悪い例>
```

### 3️⃣ 前後比較（改善の方向性を示す）

前後比較は「この状態からこの状態へ改善してほしい」という改善の方向性を具体的に示します。

:::syntax 前後比較の基本構造

```markdown
[コンポーネント名]を改善してください。

改善例：
<改善前>
[現在のUIの問題点を説明]
- デザインの問題
- ユーザビリティの問題
- アクセシビリティの問題
</改善前>

<改善後>
[改善後の理想的な状態を説明]
- 解決された問題点
- 向上したユーザー体験
- 新たに追加された機能
</改善後>
```

:::

**実践例：**
```
ユーザー登録フォームを改善してください。

改善例：
<改善前>
全ての項目が必須入力で、エラー表示が分かりにくい
パスワード要件がフォーム送信後しか表示されない
モバイル表示でラベルと入力欄が重なる
送信ボタンの状態変化がない
</改善前>

<改善後>
必須項目だけがマークされ、任意項目は明確に区別
リアルタイムバリデーションでパスワード要件を即時表示
モバイルではスタッキングレイアウトに最適化
送信中はローディング状態を表示、完了は成功メッセージ
</改善後>
```

## 💻 コードレベルでの例示

UIの見た目だけでなく、コードレベルでの例示も重要です。Reactコンポーネントの構造やCSSクラスの命名規則を指定することで、より一貫性のあるコードを生成できます。

### Reactコンポーネント構造の例示

```
ユーザープロフィールカードのReactコンポーネントを作成してください。

例：
<良い例の構造>
コンポーネント名：UserProfileCard
Props：user（オブジェクト）、size（'small' | 'medium' | 'large'）
子コンポーネント：Avatar, UserName, UserStats, ActionButton
スタイリング：Tailwind CSSクラスを使用
状態管理：useStateでフォロー状態を管理
イベントハンドラ：onFollow, onUnfollow
</良い例の構造>

コード例：
```tsx
interface UserProfileProps {
  user: {
    id: string;
    name: string;
    avatar: string;
    bio: string;
    followers: number;
    following: number;
  };
  size?: 'small' | 'medium' | 'large';
}

const UserProfileCard: React.FC<UserProfileProps> = ({ user, size = 'medium' }) => {
  const [isFollowing, setIsFollowing] = useState(false);

  const handleFollow = () => {
    setIsFollowing(true);
  };

  return (
    <div className={`bg-white rounded-lg shadow-sm p-6 ${sizeClasses[size]}`}>
      <Avatar src={user.avatar} alt={user.name} size={size} />
      <UserName name={user.name} />
      <UserStats followers={user.followers} following={user.following} />
      <ActionButton
        isFollowing={isFollowing}
        onClick={isFollowing ? handleUnfollow : handleFollow}
      />
    </div>
  );
};
```
```

### アクセシビリティの例示

アクセシビリティを考慮した例示も重要です。

```
アクセシブルなモーダルダイアログを作成してください。

例：
<アクセシビリティ要件>
役割：role="dialog" aria-modal="true"
タイトル：aria-labelledbyで関連付け
説明：aria-describedbyで関連付け
フォーカストラップ：ダイアログ内で循環
キーボード操作：ESCで閉じる、Tabで移動
スクリーンリーダー：状態変化を通知
</アクセシビリティ要件>

コード例：
```tsx
const Modal = ({ isOpen, onClose, title, children }) => {
  const modalRef = useRef<HTMLDivElement>(null);

  useEffect(() => {
    if (isOpen) {
      // フォーカストラップの設定
      const focusableElements = modalRef.current?.querySelectorAll(
        'button, [href], input, select, textarea, [tabindex]:not([tabindex="-1"])'
      );
      const firstElement = focusableElements?.[0] as HTMLElement;
      firstElement?.focus();

      // 背景のスクロールを防止
      document.body.style.overflow = 'hidden';
    }

    return () => {
      document.body.style.overflow = 'unset';
    };
  }, [isOpen]);

  if (!isOpen) return null;

  return (
    <div className="fixed inset-0 bg-black bg-opacity-50 flex items-center justify-center">
      <div
        ref={modalRef}
        role="dialog"
        aria-modal="true"
        aria-labelledby="modal-title"
        className="bg-white rounded-lg p-6 max-w-md w-full mx-4"
      >
        <h2 id="modal-title" className="text-xl font-semibold mb-4">
          {title}
        </h2>
        <div className="mb-6">
          {children}
        </div>
        <button
          onClick={onClose}
          className="px-4 py-2 bg-gray-200 text-gray-800 rounded hover:bg-gray-300"
        >
          閉じる
        </button>
      </div>
    </div>
  );
};
```
```

## 🛠️ Few-shotプロンプトを動かして確認してみよう

:::step

1. ハンズオン用プロジェクトの準備

任意の場所（デスクトップなど）で`v0-few-shot-practice`フォルダを作成し、VSCodeで開いてください。

_フォルダ作成コマンド実行_
```bash

cd ~/Desktop/v0-few-shot-practice
code .
```

1. 基本プロンプトでのUI生成

まずは基本プロンプトでユーザープロフィールカードを生成してみましょう。

## 📊 Few-shotプロンプトの効果測定

few-shotプロンプトの効果を定量的に測定する方法を学びましょう。

### 品質メトリクスの定義

コンポーネントの品質を測定するためのメトリクスを定義します。

```
# UIコンポーネント品質評価シート

## デザイン品質（25点）
- 視覚的一貫性（5点）：ブランドガイドラインへの準拠度
- カラーコントラスト（5点）：アクセシビリティ基準の満足度
- スペーシング（5点）：余白の適切さと一貫性
- タイポグラフィ（5点）：文字の読みやすさと階層性
- レイアウト（5点）：要素の配置とバランス

## 機能品質（25点）
- インタラクション（5点）：ユーザー操作の反応性
- レスポンシブ（5点）：異なる画面サイズでの表示品質
- パフォーマンス（5点）：表示速度と動作の軽快さ
- アクセシビリティ（5点）：障碍者への配慮
- エラー処理（5点）：エラー時の適切な表示

## コード品質（25点）
- 構造（5点）：コンポーネントの適切な分割
- スタイル（5点）：CSSの整理と再利用性
- 状態管理（5点）：データフローの明確さ
- テスト（5点）：テストコードの品質
- ドキュメント（5点）：コードコメントと説明

## 再現性（25点）
- 一貫性（10点）：同じプロンプトでの再現性
- 安定性（10点）：異なる環境での安定性
- 保守性（5点）：将来の変更への対応容易さ
```

### A/Bテストの実施

基本プロンプトとfew-shotプロンプトのA/Bテストを実施します。

```
# Few-shotプロンプトA/Bテスト計画

## テスト対象
- コントロールグループ：基本プロンプト
- テストグループ：Few-shotプロンプト

## テスト期間
- 期間：2週間
- サンプル数：各グループ20件のUI生成

## 評価項目
1. 生成時間の比較
2. 手動修正回数の比較
3. 品質スコアの比較
4. ユーザビリティテスト結果

## 評価方法
- 自動評価：コード品質チェックツール
- 手動評価：デザイナーとエンジニアによるレビュー
- ユーザーテスト：実際のユーザーによる使用感評価

## 成功基準
- 品質スコア：20%以上の向上
- 修正時間：30%以上の削減
- ユーザー満足度：15%以上の向上
```

### 継続的な改善

測定結果に基づいて、プロンプトを継続的に改善します。

```
# プロンプト改善サイクル

## 1. 現状分析
- 現在のプロンプトの問題点を特定
- 生成結果の品質を測定
- ユーザーからのフィードバックを収集

## 2. 改善策の立案
- 問題点に対する解決策を検討
- 新しい例示パターンを考案
- A/Bテストの計画を立案

## 3. 実装とテスト
- 改善版プロンプトを実装
- A/Bテストを実施
- 結果を分析

## 4. 標準化と展開
- 効果的な改善策を標準化
- チーム全体に展開
- ドキュメントを更新

## 5. モニタリング
- 導入後の効果をモニタリング
- 新たな問題点を発見
- 次の改善サイクルへ
```

## まとめ

Few-shotプロンプトは、v0で高品質なUIを一貫して生成するための強力な技術です。具体的な例を示すことで、AIモデルの理解を深め、望ましい結果を再現可能に生成できます。

:::note 要点のまとめ

- Few-shot learningは少数の具体例を示すことでAIの生成品質を向上させる手法
- ポジティブ例、ネガティブ例、前後比較など目的に応じて例示を使い分ける
- UIコンポーネント別に最適な例示パターンがある
- コードレベルでの例示も重要で、アクセシビリティや保守性を向上させる
- 例示ライブラリを構築し、チームで共有することでプロジェクト全体の品質を向上させる
- 定量的な効果測定と継続的な改善が重要

:::

次のステップとして、**[差分指示（部分更新/セレクタ指定）](./diff-prompts.md)**を学ぶことをお勧めします。diffプロンプトを組み合わせることで、既存のUIを効率的に改善し、より精密なコントロールが可能になります。

Few-shotプロンプトは、プロンプト設計スキルの中核をなす重要な技術です。実際のプロジェクトで積極的に活用し、チームのUI生成品質を向上させていきましょう。
