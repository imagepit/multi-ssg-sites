プロンプトはAIへの「設計図」です。適切に設計されたプロンプトは、開発者が意図した通りのUIを生成し、ブランドの一貫性を保ち、チーム全体の開発効率を劇的に向上させます。

## プロンプト設計の重要性

### プロンプトとは何か？

プロンプトはAIへの**指示書**であり、設計図です。v0のようなAIツールでは、プロンプトの質が直接生成されるUIの品質に影響します。適切なプロンプト設計は、以下のようなメリットをもたらします。

- **一貫性のある出力**: 同じスタイルガイドラインに基づいたUIコンポーネントの生成
- **開発効率の向上**: 手戻りの減少と最初から正しい方向での開発
- **チームコラボレーションの強化**: 共通のプロンプトパターンによる品質基準の統一
- **ブランド体験の維持**: 複数の開発者が関わっても一貫したブランド表現

:::note プロンプトの品質とは

プロンプトの品質とは、AIが**開発者の意図を正確に理解し、期待通りの結果を生成できる確率**を指します。高品質なプロンプトは具体的で、構造化され、必要なコンテキストをすべて含んでいます。

:::

### 従来の開発との比較

従来のフロントエンド開発では、デザイナーがデザインカンプを作成し、開発者がそれをコードに実装するという分業が一般的でした。しかし、v0のようなAIツールを使用する場合、**プロンプト設計が新たな「デザイン工程」**となります。

_従来の開発フロー_
```
要件定義 → デザインカンプ作成 → コーディング → レビュー → 修正
```

_v0を活用した開発フロー_
```
要件定義 → プロンプト設計 → AI生成 → レビュー → プロンプト改善
```

この変化により、開発者は**プロンプト設計スキル**を習得することが必須となります。

## このセクションで学ぶこと

このセクションでは、v0の基礎操作をマスターした開発者が、より高度なUI生成を実現するためのプロンプト設計テクニックを体系的に学びます。

### 📋 目的→制約→受入基準の書き方

構造化されたプロンプト設計の基本を学びます。明確な目的設定、適切な制約条件、具体的な受入基準を定義することで、AIが期待通りの結果を生成する確率を高めます。

**学習内容**: 目的の明確化、制約条件の設定、受入基準の具体化、プロンプトの構造化テクニック

### 🎯 ブランド/トーン/デザインシステムの指示

ブランドの一貫性を維持するためのプロンプト設計技術を学びます。カラーパレット、タイポグラフィ、コンポーネントガイドラインなどを効果的にAIに伝える方法を習得します。

**学習内容**: ブランドガイドラインのプロンプト化、トーンアンドマナーの指示方法、デザインシステムの適用テクニック

### 🔄 Few-shot/例示で品質を固定化

例示を用いたプロンプト設計（Few-shot Learning）の実践的な活用法を学びます。良い例と悪い例を示すことで、AIが期待する品質レベルを理解し、一貫性のある出力を生成するためのテクニックを習得します。

**学習内容**: Few-shotパターンの基本、品質固定化のテクニック、例示の選び方と効果的な提示方法

### 🔄 差分指示（部分更新/セレクタ指定）

既存のコードやUIの一部を効率的に更新するための差分指示技術を学びます。セレクタ指定や部分更新のプロンプト設計により、大規模な再生成を避け、開発効率を向上させる方法を習得します。

**学習内容**: セレクタ指定の基本、部分更新のプロンプト設計、効率的な差分指示のパターン

### 🚫 NG指定/否定プロンプトの使いどころ

AIに避けてほしい要素やパターンを明示的に指示する否定プロンプトの活用法を学びます。不必要な生成を防ぎ、期待する結果に近づけるためのテクニックを習得します。

**学習内容**: 否定プロンプトの基本、適切な使用場面、効果的なNG指定のパターン

## プロンプト設計を動かして確認してみよう

それでは、実際にプロンプト設計の基本を体験してみましょう。ここでは、同じ要件でもプロンプトの質によって生成結果がどのように変わるかを確認します。

:::step

1. 基本的なプロンプトで生成

まずは簡単なプロンプトでボタンコンポーネントを生成してみましょう。

```bash
# v0で以下のプロンプトを実行
ボタンを作成してください
```

2. 構造化されたプロンプトで生成

次に、目的・制約・受入基準を明確にしたプロンプトで生成します。

```bash
# v0で以下のプロンプトを実行
目的：ユーザーが重要な操作を確認するためのプライマリボタンを作成する
制約：
- ブランドカラーは青系（#2563eb）を使用
- ホバー時に視覚的フィードバックを提供
- アクセシビリティ対応（ARIAラベル含む）
受入基準：
- ボタンサイズは中（padding: 0.5rem 1rem）
- 角丸は0.375rem
- フォントはInter、フォントサイズは0.875rem
- ホバー時は背景色が暗くなる（#1d4ed8）
```

3. 結果の比較

2つのプロンプトで生成されたボタンを比較し、構造化されたプロンプトの方がより具体的で要件に合った結果が得られることを確認します。

:::

このハンズオンでわかるように、**プロンプトの質が直接生成結果の品質に影響**します。構造化されたプロンプト設計を習得することで、期待通りのUIをより効率的に生成できるようになります。

## 学習の進め方

このセクションは、各トピックが独立しながらも相互に関連し合っています。効果的な学習のために、以下の順序で進めることをおすすめします。

1. **目的→制約→受入基準**: プロンプト設計の基本を習得
2. **ブランド/トーン/デザインシステム**: 一貫性のあるUI生成を学習
3. **Few-shot/例示**: 品質の固定化と再現性向上を習得
