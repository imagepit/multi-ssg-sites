## 📚 アプリ組み立ての全体像

### Next.js + v0の開発フロー

Next.jsとv0を組み合わせることで、以下のような開発フローを実現できます：

1. **v0でコンポーネント生成** → 必要なUIコンポーネントを迅速に作成
2. **Next.jsでアプリ構造設計** → ページルーティングやレイアウトを定義
3. **コンポーネント統合** → v0コンポーネントをアプリケーションに組み込み
4. **機能実装** → データ取得、認証、状態管理などを追加
5. **デプロイメント** → Vercelなどのプラットフォームに公開

## 🏗️ アプリケーションアーキテクチャの基本

### Next.js App Routerの理解

Next.js 13以降で導入されたApp Routerは、以下の特徴を持っています：

:::note App Routerとは

React Server Componentsをベースにした新しいルーティングシステムで、サーバーサイドレンダリングの最適化、 nested routes（ネストされたルート）、layout system（レイアウトシステム）などの機能を提供します。

:::

### v0コンポーネントとの統合パターン

v0で生成したコンポーネントをNext.jsアプリケーションに統合する際の主なパターン：

1. **Client Componentsとしての利用**：インタラクティブなUIコンポーネント
2. **Server Componentsでの利用**：静的な表示コンポーネント
3. **カスタマイズと拡張**：プロジェクト固有の要件に合わせた修正

### プロジェクト構成のベストプラクティス

```
my-app/
├── src/
│   ├── app/                 # Next.js App Router
│   │   ├── layout.tsx      # ルートレイアウト
│   │   ├── page.tsx        # ホームページ
│   │   ├── (routes)/       # グループ化されたルート
│   │   └── api/           # APIルート
│   ├── components/        # カスタムコンポーネント
│   │   ├── ui/           # v0から生成したUIコンポーネント
│   │   └── layout/       # レイアウトコンポーネント
│   ├── lib/              # ユーティリティ関数
│   ├── hooks/            # カスタムフック
│   └── types/            # TypeScript型定義
├── public/               # 静的ファイル
└── package.json
```

## 🛠️ 開発環境の準備

### 必要なツールと設定

この章のハンズオンを進めるにあたり、以下の環境が必要です：

- **Node.js** 18.x 以上
- **Next.js** 14.x 以上
- **TypeScript**
- **Tailwind CSS**
- **v0アカウント**

### プロジェクトのセットアップ

:::step

1. Next.jsプロジェクトの作成

```bash
npx create-next-app@latest my-app --typescript --tailwind --app
cd my-app
```

2. v0コンポーネントの準備

v0で生成したコンポーネントを`src/components/ui/`ディレクトリに配置します。

3. 依存パッケージのインストール

```bash
npm install @radix-ui/react-icons lucide-react class-variance-authority clsx tailwind-merge
```

:::
