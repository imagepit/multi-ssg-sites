# 技術ドキュメンテーションサイトのハンズオンのコードの動作確認

コンテンツファイル **$ARGUMENTS** の技術ドキュメンテーションサイトのハンズオンのコードの動作確認します。

## コンテキスト検証

### コンテンツファイル

- $ARGUMENTS

## タスク: 技術ドキュメンテーションサイトのハンズオンのコードの動作確認

コンテンツファイルに基づいて、技術ドキュメンテーションサイトのハンズオンのコードの動作確認します。

### 1. 現在のコンテンツファイルのハンズオン内容確認

コンテンツファイルを読み込み、現在のハンズオン内容を確認：

**確認項目:**

- ハンズオンの手順・内容の確認

### 2. ハンズオンの確認用のプロジェクトフォルダの作成

コンテンツファイルに基づいて、ハンズオンの確認用のプロジェクトフォルダを作成：

**作成対象:**

- ハンズオンの確認用のプロジェクトフォルダを作成
- 作成場所は.handson/{$ARGUMENTSから「.md」を除いた名前}とする

### 3. ハンズオンの確認用のコードの作成

コンテンツファイルに基づいて、ハンズオンの確認用のコードを作成：

**作成対象:**

- **MUST** **必ずハンズオン内容に従ってコードを作成すること（ハンズオンに記載していない内容は作成しないこと！！）**

### 4. ハンズオンの確認用のコードの動作確認

ハンズオンの確認用のコードを実行し、動作確認を行います。

**確認対象:**

- ハンズオンの確認用のコードの動作確認
- ハンズオンの動作確認で必要な環境（ツール、ライブラリ、ファイル）がある場合はその環境をDockerfileで作成し、コンテナで動作確認を行う

### 5. ハンズオンの確認用のコードの動作確認結果の確認

ハンズオンの確認用のコードの動作確認結果をユーザーに連絡します。また動作確認結果からコンテンツファイルの修正要否をユーザーに連絡します。

**タスク内容:**

- 動作確認結果は.handson/{$ARGUMENTSから「.md」を除いた名前}/result.mdに保存する
  - コンテンツファイルに記載されたハンズオン内容で正常動作したかどうかを記述する
  - 動作確認結果以外にもハンズオンの記載内容でわかりづらい内容があったかどうかを記述する
- 確認結果ファイルを作成したらユーザーに連絡する

_連絡内容例_

```markdown
ハンズオンの確認用のコードの動作確認結果を確認しました。

## 動作確認結果概要

[正常動作 or エラー発生]

## 動作確認結果詳細

[動作確認結果詳細]

## コンテンツファイルの修正要否

[コンテンツファイルの修正要否]

## コンテンツファイルの修正内容

[コンテンツファイルの修正内容]
```

### 6. （コードがうまく動かない場合）ハンズオンの確認用のコードの修正

ハンズオンの確認用のコードの動作確認結果に問題がある場合は、コードを修正します。

**修正対象:**

- ハンズオンの確認用のコードの動作確認結果に問題がある場合は、コードを修正する
- コードの修正内容は.handson/{$ARGUMENTSから「.md」を除いた名前}/modified.mdに保存する

### 7.　（ハンズオンの確認用のコードの修正した場合）コンテンツファイルの修正

ハンズオンの確認用のコードの修正した場合は、コンテンツファイルを修正します。

**修正対象:**

- ハンズオンの確認用のコードの修正した場合は、正常に動くハンズオンのコード内容に修正する

## 指示

1. **コンテンツファイルの確認** - コンテンツファイルから現在のハンズオン内容を確認
2. **ハンズオンの確認用のプロジェクトフォルダの作成** - ハンズオンの確認用のコードを作成
3. **ハンズオンの確認用のコードの作成** - ハンズオンの確認用のコードを作成
4. **ハンズオンの確認用のコードの動作確認** - ハンズオンの確認用のコードを動作確認
5. **ハンズオンの確認用のコードの動作確認結果の確認** - ハンズオンの確認用のコードの動作確認結果を確認
6. **（コードがうまく動かない場合）ハンズオンの確認用のコードの修正** - ハンズオンの確認用のコードの動作確認結果を確認
7. **（ハンズオンの確認用のコードの修正した場合）コンテンツファイルの修正** - ハンズオンの確認用のコードの動作確認結果を確認

生成されたハンズオンのコードは、技術ドキュメンテーションサイトのコンテンツの品質を向上させるためのものです。
