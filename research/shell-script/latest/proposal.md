# 技術ドキュメンテーションサイト構築提案

## サイト概要

本サイトは、初心者から中級者までを対象に、シェルスクリプトの体系的な知識と実践的なスキルを提供することを目的とします。Web上に散在する断片的な情報を整理し、「入門」「リファレンス」「実践レシピ」「ベストプラクティス」といった構造化されたコンテンツを提供することで、学習者が効率的にスキルを習得できるプラットフォームを目指します。

## ターゲットユーザー

- **インフラ/サーバーサイドエンジニア**: 日々の運用業務やデプロイ作業を自動化したいエンジニア。
- **Web開発者**: ビルドプロセスや開発環境のセットアップを効率化したい開発者。
- **学生・初学者**: Linuxやコマンドラインの基礎を学び、次のステップに進みたいと考えている学習者。

- `シェルスクリプト ファイル操作`, `シェルスクリプト テキスト処理`, `シェルスクリプト 自動化`
- `シェルスクリプト best practice`, `堅牢なシェルスクリプト`
- `(コマンド名) 使い方` (例: `grep 使い方`, `sed 使い方`)

## コンテンツ構成

1.  **基礎**:
    - シェルスクリプトとは？
    - ファイルの作成と実行方法
    - 基本的なコマンド（`echo`, `ls`, `cd`, `cat`など）

2.  **コア文法（リファレンス）**:
    - 変数（定義、参照、特殊変数）
    - 制御構文（`if`, `case`, `for`, `while`）
    - 関数
    - 配列
    - 引用符（シングル、ダブル、バッククォート）の完全ガイド

3.  **実践レシピ（逆引き）**:
    - **ファイル・ディレクトリ操作**:
      - 複数ファイルを一括リネームする
      - 特定の条件でファイルを検索する
      - バックアップを作成する
    - **テキスト処理**:
      - ログファイルからエラー行を抽出する
      - CSVファイルから特定の列を抜き出す
      - 複数のファイルの内容を置換する
    - **プロセス・システム管理**:
      - プロセスを監視し、死んだら再起動する
      - ディスク使用量をチェックして通知する
    - **Web・API連携**:
      - `curl`でAPIを叩いて結果を`jq`でパースする

4.  **ベストプラクティス**:
    - **堅牢なスクリプト**:
      - `set -euxo pipefail` の解説
      - エラーハンドリングと終了コード
      - 一時ファイルの安全な扱い方
    - **可読性とメンテナンス性**:
      - スタイルガイド（命名規則、インデント）
      - 関数化のすすめ
      - コメントの書き方
    - **ポータビリティ**:
      - `bash`と`sh`の違い
      - 互換性の高い書き方
    - **テスト**:
      - `shellcheck`による静的解析
      - `bats-core`を使った単体テスト入門
